MONSTER MOVIE

Monster Movie is an interactive, audio-reactive application built by Tony Lim and Mehan Jayasuriya and used for the live performance GOEMUL, which took place at the IAC Building in NYC on March 1, 2013. This application uses Processing/Java and MAX/MSP to sync the movement of animated characters to live music. Built as part of Daniel Shiffman's 2012 Big Screens class at NYU ITP.

The codebase for Monster Movie is built on top of Shiffman's Most Pixels Ever framework. For information on how the MPE sections of this code work, see the excellent MPE documentation here.

A few notes on how this all works:

  • This application was designed to be run at a resolution of 11520 x 1080, on a series of screens powered by three seperate machines. It can also be scaled and run locally on one machine, using the "local" and "scale" parameters. To run the application (either locally or on a remote server), you'll need to first run the MPE Server application.
  • All of the animations were produced using the open-source animation software Animata.
  • All of the animation files are saved as .nmt files, which are saved in the "data" folder, alongside their associated source images.
  • We used the Processing 1.5.1 core, since that is the latest version of Processing that the Animata library for Processing is compatible with.
  • We used a MAX patch to simultaneously route commands from two MIDI controllers to Ableton Live and our application, using OSC messages. In theory, the animations in our application could be synced with virtually any live or pre-recorded music by rerouting the OSC messages in the MAX patch.
  • To see how we're receiving the OSC messages and then routing them to the machines controlling the various screens, see src>mpe>controller>Messenger.java.
  • The bulk of our code is in src>mpe>examples>Drawer.java. This is where we parse the OSC messages recieved and draw elements on the screen.
